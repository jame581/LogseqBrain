# v0.8.0 — Graph hygiene, deepening (design)

**Date:** 2026-06-23
**Status:** designed (not yet implemented)
**Theme:** deepen the graph-hygiene work shipped in the first half of v0.8.0.
**Builds on:** `docs/superpowers/specs/2026-06-23-v0.8.0-design.md` (brain-doctor + compose-time invariants).

## Motivation

The first half of v0.8.0 added the `brain-doctor` skill (5 issue classes) and compose-time invariants in `skills/_shared/logseq-format.md`. Two gaps remain:

1. **Detection is shallow.** brain-doctor catches the 5 corruption classes found in the audit, but not the broader data-quality problems a long-lived brain accumulates: malformed properties, broken internal links, duplicate entries, structural gaps.
2. **Prevention is passive.** The invariants are documented, but `brain-save` doesn't verify its own output against them before writing — it relies on the model remembering. A backstop check would catch slips at the source.

Both gaps share a root need: **one rule set, two consumers** — brain-doctor (reactive scan) and brain-save (proactive self-check). Today the rules live in two places (`logseq-format.md` narrative + `brain-doctor/references/checks.md` detection). Adding more rules and a second consumer without consolidating guarantees drift.

## Decision

**Approach A — shared rule catalog.** Promote `brain-doctor/references/checks.md` to a canonical `skills/_shared/hygiene-rules.md`. Both skills read it. Each rule is a structured entry; two fields (`auto-fixable`, `enforced-at`) are what the consumers branch on.

Rejected: B (add checks independently → rules in three places, drift); C (also extract a `_shared/self-check.md` procedure → over-engineered for a single consumer, YAGNI).

## Architecture

```
skills/_shared/
  hygiene-rules.md      NEW. Canonical catalog (absorbs checks.md + 4 new rules).
  logseq-format.md      keeps narrative compose-time invariants; links to catalog
                        instead of re-listing detection/remediation.
skills/brain-doctor/
  SKILL.md              scans/fixes by iterating skills/_shared/hygiene-rules.md
  references/checks.md  REMOVED (content promoted to _shared).
skills/brain-save/
  SKILL.md              new self-check step (4.5) over the compose+auto-fixable
                        subset of the catalog.
```

### Rule entry schema (`hygiene-rules.md`)

Each rule is one entry with:

- `id` — stable slug.
- `severity` — `breaks-render` | `phantom-page` | `data-quality`.
- `enforced-at` — `compose` (brain-save can prevent it) and/or `scan` (brain-doctor finds it).
- `auto-fixable` — `yes` | `safe-only` | `report`.
- `detection` — the grep/procedure.
- `remediation` — the transform, or the "report with suggestion" guidance.

## The 4 new rules

### `malformed-property` — `safe-only`
- **What/why:** single-colon `key: value` where Logseq needs `key::` → silently not a property, so it disappears from the queries `brain-status` / `brain-load` rely on.
- **Detection (high-precision):** only a **known** property key in bullet-leading position with a single colon:
  `^(\s*-?\s*)(status|last-updated|project|projects|tags|type|context|rationale|decision|open-questions|resume-here|files-modified|skills-used|related-tickets):(\s|$)`
  Deliberately ignores prose ("Root cause: …"), times ("18:18"), URLs, ratios. The exact key list is pinned against `skills/brain-init` page templates at implementation time.
- **Remediation:** `key:` → `key::`. Auto-fix is safe **because** the key is from the brain's own vocabulary and at bullet start.

### `broken-link` — `report`
- **What/why:** any internal `[[Target]]` (excluding `file://`) whose page file doesn't exist.
- **Detection:** real-page set (filenames `___`→`/`) vs. all `[[…]]` targets → the difference. Sub-classify each:
  - missing namespace (e.g. `[[CRMGM-x]]`) → handled by `unnamespaced-link` (auto-fix).
  - fuzzy-close to an existing page → likely typo → report **with the suggested match**.
  - no close match → forward-reference → report under "intentional? leaving as-is."
- **Remediation:** report with suggestion; never auto-delete a link.

### `duplicate-entry` — `report`
- **What/why:** the same bullet appended twice under `## Session Log` (project pages) or `## Activity` (journals), from a repeated save.
- **Detection:** within those sections only, normalize each bullet (strip leading `HH:mm`/date prefix + indentation) and flag **exact** repeats; optionally surface ≥0.9-similar pairs as "possible".
- **Remediation:** report the pair(s); user chooses which to drop. No auto-removal.

### `structural-integrity` — `report`
- **What/why:** a `Projects___*` / `Tasks___*` page missing required properties (the ones `brain-init` seeds, e.g. `status::`, `last-updated::`), or a section that is empty / placeholder-stub-only.
- **Detection:** check each page's property block (offset 0) for required keys; flag `## Heading`s whose only child is an italic `_stub_` or nothing.
- **Remediation:** report. One **optional** safe suggestion: backfill a missing `last-updated::` from the newest Session Log date (offered, not auto-applied).

## brain-save write-time self-check (step 4.5)

- Between "prepare updates" (step 4) and "write" (step 5): scan **the composed block text** against the `enforced-at: compose` + auto-fixable rules — `code-in-braces`, `bare-hash-tag`, `unnamespaced-link`, `file-link`, `malformed-property`.
- **Silently self-correct** to the invariant form before the Edit — it's the skill's own output; no prompt.
- Pure in-memory check on the composed block only — **no extra file reads**, no re-scan of the page. A cheap backstop, not a second pass.
- Does **not** run the report-only rules (`broken-link`, `duplicate-entry`, `structural-integrity`, `description-link`) — those need whole-graph context and judgment, which stays brain-doctor's job. Self-check's contract: *never emit a mechanical violation.*
- `journey-log` already writes namespaced `[[Projects/X]]` links by template and stays compliant — no self-check added there.

## brain-doctor flow & report

Existing flow unchanged (Scan → Report → Confirm → Backup → Repair → Verify → journey-log). Changes:

- **Scan** iterates all 9 catalog rules where `enforced-at: scan`.
- **Report** groups by `auto-fixable`:
  ```
  Brain health: 12 issues across 6 pages
  Auto-fixable (5):  3 {{code}}, 1 bare #tag, 1 malformed property
  Needs your call (7): 2 broken links (1 likely typo → suggests [[Tasks/CRMGM-1982]]),
                       3 duplicate Session Log entries, 2 pages missing last-updated::
  ```
- **Fix:** auto-apply the "safe" group on one confirmation; walk the "needs your call" group with per-item (or batch) approval — applying only what the user OKs.
- **Verify:** re-run detections + per-file backtick parity, as today.
- **Out of scope:** no `brain-status` health line, no scheduled/auto scan (automation was explicitly deferred). Clean future add.

## Validation (manual — no runtime/tests in this repo)

1. Seed a scratch graph with one instance of each of the 9 issues. `brain-doctor` report → counts match a manual grep; grouping (auto-fixable vs needs-call) is correct.
2. Fix mode → only `malformed-property` (+ the 4 existing auto-fix rules) write without per-item confirmation; the 3 report-only new rules never write unprompted. Re-scan clean; backtick parity even.
3. `brain-save` round-trip where the composed text contains a `{{x}}`, a `#44`, and a `key:` → step 4.5 corrects all three before the Edit; the written page has none.
4. `malformed-property` precision: a page with prose "Root cause: foo", "18:18", and a real `status: active` → only the last is flagged.
