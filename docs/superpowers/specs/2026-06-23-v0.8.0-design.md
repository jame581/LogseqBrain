# v0.8.0 — Graph hygiene (design)

**Date:** 2026-06-23
**Status:** shipped
**Theme:** detect, repair, and prevent the format mistakes that silently corrupt a Logseq graph.

## Motivation

An audit of a real, heavily-used ClaudeBrain graph (49 pages + 49 journals) surfaced pervasive corruption that had been generated by `brain-save` itself, invisible until the graph is opened in Logseq:

| Issue | Count | Effect in Logseq |
|---|---|---|
| Inline code wrapped in `{{ }}` | ~1,031 spans, 16 files | broken/empty macro (`:macros {}` is the default) |
| Bare `#number` / hex-color tags | 125 | phantom empty pages (`44`, `0066CC`, …) — the "empty 2–3-char pages" |
| Un-namespaced `[[CRMGM-x]]` links | 16 | phantom duplicate of an existing `Tasks/CRMGM-x` page |
| `[[file:///…]]` links | 7 | phantom page titled with the whole path |
| Junk / description / typo links | 7 | junk pages (`[[link]]`, `[[Projects/Unicorn]]`, …) |

Root cause: the format guidance (`CLAUDE.md` invariants, `skills/_shared/logseq-format.md`) covered filenames/bullets/properties/dates and *edit-time* normalization, but said nothing about **compose-time** content generation — so every save risked re-introducing the above.

This is also the v0.8.0 the roadmap had been missing: a concrete theme that does **not** depend on Logseq's external roadmap (unlike the deferred DB-API / CLI-sync items).

## What shipped

### Prevention (compose-time invariants)
- New "Content-generation invariants" section in `skills/_shared/logseq-format.md`:
  1. Inline code uses backticks, **never** `{{ }}` (double-backtick fence when the snippet contains a backtick).
  2. Escape `#` before a number or hex color (`PR `#44``); real tags are `#[[Page Name]]`.
  3. Task/project links must be namespaced (`[[Tasks/CRMGM-1234]]`, `[[Projects/Name]]`).
  4. Local file paths are markdown links `[label](file:///…)` or backticks — never `[[file://]]`.
  5. Don't bracket a description or an internal memory slug.
- `brain-save/SKILL.md` step 4 now points at these invariants when composing content.
- `CLAUDE.md` "Logseq format invariants" list extended with the same rules.

### Cure (the `brain-doctor` skill)
- `skills/brain-doctor/SKILL.md` — orchestrator with **Report** (read-only) and **Fix** (backup → confirm → repair → verify) modes.
- `skills/brain-doctor/references/checks.md` — catalog of the 5 issue classes, each with a detection grep and a remediation transform; documents the edge cases (spans with literal braces / backticks) that need hand-fixing.
- Design choices:
  - **Report before repair, backup before write, confirm before both** — consistent with the plugin's "never persist without confirmation" rule, and the graph is usually not git-tracked.
  - **Bulk transform is acceptable here** even though the save path is surgical — `brain-doctor` is a one-time maintenance operation, not a per-session conflict-sensitive write, and it always backs up first.
  - **A missing-page link is not always a bug** — forward-references are surfaced but left alone unless the user opts in.

### One-time cleanup
The maintainer's own graph was repaired as part of this work (counts in the table above), with a backup taken first. Verified: 0 broken macros remain, all backtick spans balanced, no live phantom tags; 4 legitimate forward-reference links (`Tasks/CRMGM-1094, 1802, 1845, 2009`) intentionally retained.

## Validation

No runtime/tests in this repo — validate by invoking against a real graph:
1. `brain-doctor` report mode on a graph with known issues → counts match a manual grep.
2. Fix mode → backup created, issues repaired, re-scan clean, backtick parity even per file.
3. A `brain-save` round-trip → composed content uses backticks / namespaced links / escaped `#`, introducing no new phantom pages.

## Not in scope (still deferred to Future)
Storage abstraction, Logseq DB plugin API, headless CLI sync, Sync conflict resolution, packaging refinements — all still gated on Logseq's own roadmap.
